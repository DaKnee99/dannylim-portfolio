CREATING A NOTIFICATION SOURCE OF TRUTH

Built a company-wide notification repository that eliminated copy inconsistencies and streamlined collaboration across design, engineering, and marketing teams.

Impact

  • Audited 60 notifications across the product
  • Created self-service system adopted by entire product team
  • Eliminated cross-team confusion and reduced implementation back-and-forth
"The repository makes it so much clearer to understand what notifications we have. Before this, I'd have to dig through code or ask around." — Product Manager

UX Writer  |  4 weeks  |  Product, Design, Engineering, Marketing

The Problem

Grof's app notifications were inconsistent—the same actions described differently across the product. Without a centralized system, designers, engineers, and marketers worked from different terminology, creating user confusion and slowing down shipping.

The Solution

A centralized notification repository with standardized templates, placeholder formats, and color-coded categorization. The system serves as a single source of truth, enabling self-service implementation without approval bottlenecks.

Repository Table — Color-Coded Tags & Placeholder Format

The Process

1. Audit Phase

Reviewed 60 notifications against four criteria:

  • Usage status (active vs. obsolete)
  • Redundancy (duplicate messaging)
  • Clarity of intent (FYI vs. action required)
  • Tone alignment (brand voice consistency)

2. Repository Design

Each notification entry includes:

  • Purpose categorization
  • Recipient type
  • Feature association
  • Template with clear placeholders (e.g., {user_name}, {transaction_amount})
  • Color-coded tags for quick scanning
Example Entry — Placeholder Format

3. Team Workflow

UX + Marketing draft copy → Add to repository → Engineering implements from templates

Result: No approval bottlenecks, self-service system.

Key Design Decisions

Why Placeholders?

Engineers know exactly which content is dynamic vs. static—no guessing, no waiting for clarification.

Why Color-Coding?

Visual system enables quick scanning across 60+ notifications to find relevant examples.

Why Self-Service?

Reduces bottlenecks while giving marketing tone consistency and engineering a clear implementation path.

What I Learned

Minor word choices have major impact on user trust. The difference between "Payment processed" and "Payment successful"—passive vs. active voice—shapes whether users feel informed or confused.

Design systems extend beyond UI components. Content systems are equally critical. A button style guide is useless if button labels are inconsistent.
